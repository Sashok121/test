Необходимо отправить форму "Обратная связь" для сохранения данных в БД и отправки на емейл администратора.

1. Перед отправкой провалидировать форму (с помощью javascript) на предмет корректности даных
ФИО - Обязательно для заполнения, любой текст до 100 символов.
E-mail - обязательно для заполения, формат емейл до 100 символов.
Телефон - не обязательно для заполнения. код телефона максимум 4 символа, тольчо чиста и знак "+".
Сам номер телефона - должна быть маска +380(__)___-__-__ только числа, длина до 20 символов.
Описание - необязательное для заполения, любой текст и числа длиной до 1000 символов.
Если валидация не проходит показать пользователю сообщение с указанем точной ошибки и подсветить поле ,в котором допущена ошибка.

2. Создать базу данных в mySql с именем site1. В ней создать таблицу mod_feedback c полями:
id - идентификатор записи, int, primary key, autoincrement.
fio - varchar 255.
email - varchar 100.
phone - varchar 30.
descr - text.
dt - дата и время создания записи в формате "YYY-MM-DD hh:mm:ss".
ip - ip-адрес, с котрого было отправлено сообщение.

3. Перед сохранение данных в БД еще раз провалидировать данные уже с помощью php, что бы в базу точно записать корректные данные.
Предусмотреть защиту от XSS-атак, mySql-иньекций.

4. Отправку формы делать методом POST. Приветствуется и добавляет скилов отправка формы аяксом.

5. Отправить уведомление на емейл администратора (пусть будет твой емейл) о новом сообщении с сайта. В теле сообщения указать все данные из заполненной формы обратной связи.

6. После сохранения данных и отправки уведомления на емейл показать страницу с результатом (или уведомление,если отправку формы делали Аяксом).
Если все прошло успешно показать "Ваше сообщение успешно отправлено".
Если во время сохранения данных в таблицу или во время отправки уведомления произошла ошибка - то на странице с результатом показать сообщение (уведомление) об этой ошибке.

Важно! Везде должна быть одна и таже кодировка (utf-8).
И для скриптов, и для html-документа, и для базы данных и таблиц в ней (utf8_general_ci).


Результаты тестового задания прислать архивом, с отдельным файлом дампа базы данных.
